# Scenario Eval Checklist

Ручная проверка use-case сценариев на тестовом Google-аккаунте.

**Требования для запуска:**
- Поднят полный стек: `docker-compose up -d`
- Telegram-бот подключён к ngrok (`TELEGRAM_WEBHOOK_URL` настроен)
- Тестовый Google-аккаунт авторизован в боте (`/start` → OAuth)
- Наблюдение за логами: `docker-compose logs -f agent`

---

## UC-001 — Текстовый ввод: создание события

**Сценарий:** Пользователь отправляет: `"Встреча с Ваней завтра в 14:00 на 1.5 часа"`

| Шаг | Ожидаемый результат |
|-----|---------------------|
| Сообщение отправлено в Telegram | Бот отвечает сообщением-подтверждением с кнопками ✅/❌ |
| Нажать ✅ Подтвердить | Бот отвечает: «Событие создано» |
| Проверить Google Calendar | Событие «Встреча с Ваней» на завтра 14:00–15:30 |
| Latency | Ответ появляется менее чем за 7 секунд |

**Проверка в Langfuse:** trace с action=`create_event`,

## UC-002 — Голосовой ввод: создание события

**Сценарий:** Пользователь отправляет голосовое сообщение с текстом: «Запланируй тренировку на субботу в 8 утра»

| Шаг | Ожидаемый результат |
|-----|---------------------|
| Аудио-файл отправлен | Бот подтверждает получение, транскрибирует |
| Бот показывает транскрипт + подтверждение | Текст распознан корректно |
| Нажать ✅ | Событие «Тренировка» создано в Google Calendar |
| Latency | Ответ с учётом ASR < 10 секунд |

**Примечание:** Аудио в формате `.ogg` (Telegram). Проверить что `ffmpeg` отрабатывает конвертацию.

## UC-003 — Конфликт расписания

**Предусловие:** В Google Calendar уже есть событие завтра с 14:00 до 15:00.

**Сценарий:** Пользователь отправляет: `"Встреча с командой завтра в 14:00"`

| Шаг | Ожидаемый результат |
|-----|---------------------|
| Сообщение отправлено | Бот сообщает о конфликте + предлагает свободные слоты кнопками |
| Отображены альтернативные слоты | Минимум 1 предложенный слот, не пересекается с занятым временем |
| Нажать на один из предложенных слотов | Бот создаёт событие в выбранном слоте |
| Проверить Google Calendar | Событие создано без конфликта |

## UC-004 — Задача с дедлайном

**Сценарий:** Пользователь отправляет: `"Создай задачу: подготовить квартальный отчёт, срок — пятница"`

| Шаг | Ожидаемый результат |
|-----|---------------------|
| Сообщение отправлено | Бот показывает подтверждение задачи с дедлайном |
| Нажать ✅ | Задача создана в Google Tasks |
| Проверить Google Tasks | Задача «Подготовить квартальный отчёт» с дедлайном в пятницу |

## UC-005 — Низкая уверенность LLM (уточнение)

**Сценарий:** Пользователь отправляет: `"создай встречу"`  (одно слово без деталей)

| Шаг | Ожидаемый результат |
|-----|---------------------|
| Сообщение отправлено | Бот задаёт уточняющий вопрос («Когда? С кем?») — НЕ создаёт событие |
| Пользователь отвечает: «завтра в 15» | Бот уточняет намерение и создаёт событие |
| Langfuse | Видна последовательность: низкий confidence → ask_user → resume |

## UC-006 — Истёкший OAuth токен

**Предусловие:** Вручную удалить/инвалидировать токен в БД или отозвать доступ в Google.

**Сценарий:** Пользователь отправляет любой запрос на создание события.

| Шаг | Ожидаемый результат |
|-----|---------------------|
| Сообщение отправлено | Бот отвечает с просьбой переавторизоваться + ссылка |
| Ссылка открывается | Редирект на Google OAuth, после — обратно в бот |
| После OAuth | Исходный запрос обрабатывается, событие создаётся |

## UC-007 — Таймаут/ошибка LLM

**Предусловие:** Временно выставить невалидный `MISTRAL_API_KEY` в контейнере.

**Сценарий:** Пользователь отправляет любой запрос.

| Шаг | Ожидаемый результат |
|-----|---------------------|
| LLM недоступен | Бот выполняет retry × 3 (exponential backoff) |
| После 3 неудач | Бот отвечает: «Не удалось обработать запрос. Попробуй чуть позже.» |
| Логи | `extract_intent_llm_error` с attempt=1,2,3 |
| Метрики Prometheus | `agent_llm_errors_total` увеличен |

## UC-008 — Граничные случаи

### 8a — Сообщение > 4000 символов
**Сценарий:** Отправить текст длиннее 4000 символов.

| Шаг | Ожидаемый результат |
|-----|---------------------|
| Сообщение отправлено | Бот отклоняет: «Сообщение слишком длинное» |
| Контент НЕ передаётся в LLM | Проверить что запроса к Mistral нет в Langfuse |

### 8b — Rate limit (5 сообщений/минута)
**Сценарий:** Отправить 6 сообщений подряд за < 60 секунд.

| Шаг | Ожидаемый результат |
|-----|---------------------|
| Первые 5 сообщений | Обрабатываются нормально |
| 6-е сообщение | Бот отвечает: «Слишком много запросов, подожди минуту» |
| Через 60 секунд | Запросы снова принимаются |
